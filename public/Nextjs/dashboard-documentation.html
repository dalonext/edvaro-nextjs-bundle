Edavro — LMS Admin Dashboard

A fully production-grade Learning Management System admin dashboard built with Next.js 16, React 19, TypeScript, Tailwind CSS 3, and Redux Toolkit. Ships with a complete design system, three-way dark/light/system theme mode, 25 pages, and zero hardcoded colors.

Installation

Extract the zip Unzip the downloaded file. You will find the edvaro-dashboard-nextjs folder inside.
Install dependencies Open a terminal inside the project folder and run: 
cd edvaro-dashboard-nextjs
npm install
Start the development server 
npm run dev

Open http://localhost:3000 in your browser. The app redirects to /login automatically.

Build for production 
npm run build
npm run start
ℹ️ This project uses mock JSON data — no backend or database is required to run it. All data lives in src/data/*.json.

State Management

The project uses Redux Toolkit with redux-persist. The store is at src/store/index.ts.

Slice State Key actions
uiSlice theme, sidebarOpen, sidebarCollapsed, mobileMenuOpen, isMobile, language, notificationSettings setTheme, setSidebarOpen, setSidebarCollapsed, setMobileMenuOpen, setIsMobile, setLanguage
authSlice user, isAuthenticated, token login, logout, setUser
courseSlice courses list setCourses, addCourse, updateCourse, deleteCourse
studentSlice students list setStudents, addStudent, updateStudent, deleteStudent
transactionSlice transactions list setTransactions, addTransaction
certificateSlice certificates list setCertificates, addCertificate
chatSlice conversations, activeConversationId setActiveConversation, sendMessage
wishlistSlice wishlisted course IDs toggleWishlist, setWishlist
dashboardSlice dashboard metrics cache setMetrics
notificationSlice notification list addNotification, markRead, clearAll
ℹ️ Theme persistence: uiSlice.setTheme writes to localStorage under the key "edavro-theme". An inline script in layout.tsx reads this key before the first paint to prevent flash of unstyled content (FOUC).

Design Tokens

Every color, spacing token, and typography value is defined as a CSS variable in src/styles/globals.css and mapped to a Tailwind utility in tailwind.config.ts. There are zero hardcoded hex values in any component. To re-brand the entire dashboard, update the CSS variables only.

How to Rebrand

Open src/styles/globals.css and change the brand variables under :root: 

/* Change these 4 values to rebrand the entire dashboard */
:root {
  --color-primary:        #743FFE;   /* buttons, active states */
  --color-primary-hover:  #7B48FF;   /* hover, filled buttons */
  --color-primary-active: #6A3DE8;   /* pressed state */
  --color-primary-light:  #b092ff;   /* subtle tints, accents */
}
✅ Changing those 4 variables updates buttons, badges, sidebar active states, chart colors, focus rings, gradient headers, and all brand accents across all 25 pages instantly.

Dark Mode

Three modes are supported: Light, Dark, and System (follows OS preference). The current mode is stored in Redux and persisted to localStorage.

🌞 Light
White surfaces, light purple dashboard backgrounds, dark text.
🌙 Dark
Deep navy backgrounds (#0d0f1a), dark card surfaces, light text. All CSS vars swap via .dark class on <html>.
💻 System
Reads prefers-color-scheme and updates in real-time if OS preference changes.

How to Switch Theme

Desktop: Theme toggle in the Topbar (top-right, sun/moon/monitor icons).

Mobile: 3-way toggle at the bottom of the Sidebar.

Programmatically:

import { setTheme } from "@/store/slices/uiSlice";
import { useAppDispatch } from "@/store/hooks";

const dispatch = useAppDispatch();
dispatch(setTheme("dark"));    // or "light" or "system"
⚠️ FOUC prevention: An inline <script> in src/app/layout.tsx runs before the first paint and applies the correct class. Do not remove this script.
