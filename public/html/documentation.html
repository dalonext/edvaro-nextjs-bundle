01 Overview

Thank you for purchasing Eadvro Dashboard. This is a modern, fully responsive PWA template for online learning platforms, course marketplaces, and instructor dashboards.

Every page is a plain HTML file — no React, no Vue, no Angular. You can open it directly in a browser or integrate it into any backend system.

  • Tailwind CSS v4 — utility-first CSS, compiled via npm
  • Pure Vanilla JS — zero runtime framework dependencies
  • Course analytics — charts for student progress, lesson completion, and learning metrics
  • 35+ HTML pages — enrollment, course list, lesson, dashboard, profile, and settings
  • Fully responsive — desktop, tablet, and mobile with a polished sidebar experience
  • Toast notifications — stackable success / error / info / warning system
  • Smart redirect — splash screen detects login state and routes automatically

03 Installation

1

Extract the ZIP

Extract the downloaded ZIP file. You will get a eadvro/ folder.

2

Open in VS Code

Open the eadvro/ folder in VS Code.

3

Install dependencies

Open the terminal in VS Code and run:

npm install
4

Start development

Run the dev server — this starts Tailwind watch + browser-sync with hot reload: 

npm run dev

Browser opens automatically at http://localhost:3000 on the login page.

04 Running the Project

Development

npm run dev

Starts two things simultaneously:

  • Tailwind CSS watcher — recompiles src/css/output.css on every change
  • Browser-sync server at http://localhost:3000 — auto-reloads on file changes

Production Build

npm run build

Builds everything into the dist/ folder:

  • Minifies CSS via Tailwind
  • Copies all JS, assets, fonts and images
  • Copies vendor libs (ApexCharts)
  • Minifies all HTML pages

Preview Production Build

npm run preview

Serves the dist/ folder at http://localhost:3000 so you can verify the build before deploying.

⚠

Always run npm run build before npm run preview. Preview serves the last built version.

08 Authentication

All authentication logic is in auth.js and uses localStorage to simulate a backend — no server required.

Login behavior

  • If a registered user exists in localStorage and email matches — password is validated
  • If a different email is used — any password works (demo / guest mode)
  • Sign In button is disabled until both fields have content

Registration flow

register → verify-email → choose-location → profile-photo → dashboard

Password reset flow

forgot-password → enter-otp → new-password → login

Connecting a real backend

Replace the Storage.set / Storage.get calls inside LoginForm and RegisterForm in auth.js with your own fetch() API calls. All UI (toasts, redirects, validation) stays the same.

11 Toast Notifications

The toast system is available as window.Toast on every page that loads layout.js or auth.js. Toasts stack, auto-dismiss after 3.5s, and have a close button. 

Toast.success("Saved!", "Your changes have been saved.");
Toast.error("Failed", "Something went wrong.");
Toast.info("Note", "Optional message body.");
Toast.warning("Warning", "Please check your input.");

12 Search & Filter

search.js creates its own DOM elements appended to <body> — no extra HTML needed. It activates automatically when these elements exist on the page: 

<input id="headerSearchInput" type="search" />
<button id="filterIconBtn"></button>

Filter options

  • Price range slider ($0 – $100)
  • Category chips (Pizza, Burger, Sushi, Drinks, Pasta, Dessert)
  • Payment type (Cash / Card / Online)
  • Star rating (1–5 stars)

Replacing mock data with real API

// In search.js — find MOCK_PRODUCTS and replace:
const MOCK_PRODUCTS = await fetch('/api/products').then(r => r.json());

13 Colors & Theming

All brand colors are defined as CSS variables in src/css/main.css. Change one value and every component updates. 

/* src/css/main.css — Tailwind @theme block */
@theme {
  --color-primary:         #7b48ff;  /* Primary violet */
  --color-primary-hover:   #6d39f0;  /* Hover / action */
  --color-primary-soft:    #743ffe;  /* Soft accent */
  --color-primary-light:   #9f79ff;  /* Light violet */
  --color-secondary:       #b092ff;  /* Secondary accent */
  --color-dark:            #050505;  /* App background */
  --color-card:            #1a1a1a;  /* Card / panel */
  --color-border:          #3b3b3b;  /* Border lines */
  --color-muted:           #8b8b8b;  /* Muted text */
}

After changing any color, run npm run dev to recompile Tailwind.
