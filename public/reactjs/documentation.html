Overview

EdaVro is a mobile-first e-learning platform built with React 19. It features authentication flows, a comprehensive learning dashboard, course management, payment processing, and user support sections. The application is built as a Progressive Web App (PWA) for offline capability and app-like experience.

Key Features

  • Authentication: Sign in, sign up, OTP verification, password reset, profile completion
  • Dashboard: Personalized home, course browsing, wishlist, search functionality
  • Courses: My courses, free courses, recommended courses, certifications, learning progress tracking
  • Payments: Secure payment gateway, card management, multiple payment methods
  • Support: Help center, FAQ, live messaging, notifications, preferences
  • PWA: Install banner, service worker, offline support, app-like experience
29+ Pages

Routing & Navigation

Routes are centrally defined in src/routes/AppRoutes.jsx with a clear separation between public and protected routes. The PrivateRoute component enforces authentication for sensitive areas.

Public Routes (No Auth Required)

Route Component Purpose
/ Pwa.jsx PWA landing page with hero section
/splash SplashScreen.jsx Splash screen on app load
/auth/signin SignIn.jsx User login
/auth/signup SignUp.jsx User registration
/documentation DocumentationPage.jsx In-app documentation

Protected Routes (Auth Required - Dashboard)

Route Component Purpose
/dashboard/home Home.jsx Main dashboard with courses and recommendations
/dashboard/search SearchBar.jsx Full search interface
/dashboard/wishlist Wishlist.jsx Saved/bookmarked courses

Progressive Web App (PWA)

EdaVro is built as a Progressive Web App for offline capability, installability, and app-like experience.

Key Features

  • Install Prompt: Smart banner that appears when app is installable
  • Offline Support: Service worker caches assets for offline functionality
  • App Icons: Custom icons for home screen and dock
  • Standalone Mode: Runs as full-screen app without browser UI
  • Fast Loading: Service worker pre-caches critical resources

Forms & Validation

Form handling uses React hooks for local state management with client-side validation.

Validation Techniques

  • HTML5 Attributes: required, type, minLength, pattern, min, max
  • Custom Functions: validateEmail(), validatePhone(), etc.
  • State Management: Track form state and errors in useState hooks
  • Real-time Feedback: Show errors as user types or after blur
